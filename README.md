# Descripción

**GesTock** es una aplicación móvil desarrollada para facilitar la gestión de inventarios en pequeños negocios. La aplicación permite registrar productos, controlar existencias, actualizar información y administrar el inventario de forma rápida e intuitiva desde un dispositivo móvil.

Su objetivo es ayudar a emprendedores y pequeños comerciantes a mantener un mejor control de sus productos, reduciendo errores y mejorando la organización del inventario.

---

# Problemática

Muchos pequeños negocios administran su inventario de forma manual mediante libretas o hojas de cálculo, lo que dificulta el control de existencias y aumenta el riesgo de pérdidas, errores en los registros y desabasto de productos.

Además, la falta de herramientas digitales limita la capacidad de los emprendedores para consultar información de manera rápida y tomar mejores decisiones sobre su inventario.

---

# Solución Propuesta

Se desarrolló una aplicación móvil que permite:

- Registrar productos.
- Editar información de los productos.
- Eliminar productos.
- Consultar el inventario.
- Controlar la cantidad disponible de cada producto.
- Agregar fotografías de los productos.
- Gestionar la información mediante operaciones CRUD.

---

# Arquitectura

La aplicación fue desarrollada bajo el patrón de arquitectura **MVVM (Model-View-ViewModel)** para mantener una correcta separación entre la lógica de negocio y la interfaz de usuario.

La arquitectura está compuesta por:

- Interfaz desarrollada con Xamarin.Forms.
- Lógica de negocio implementada en C#.
- Base de datos local SQLite.
- Navegación mediante FlyoutPage.

---

# Desarrollo del Proyecto

El desarrollo comenzó con el análisis de las necesidades de pequeños negocios respecto al control de inventarios.

Posteriormente se diseñó la base de datos para almacenar la información de los productos, incluyendo nombre, descripción, precio, cantidad disponible e imagen.

Se implementó la aplicación utilizando Xamarin.Forms, desarrollando los módulos de registro, consulta, edición y eliminación de productos.

Finalmente se realizaron pruebas funcionales para validar el correcto funcionamiento del sistema y garantizar la integridad de la información almacenada.
